# focus-timer 仕様書

## 1. 目的

Windows 上で常駐的に利用できる、小型のフローティング式フォーカスタイマーを提供する。  
タイマー本体とは別にタスク管理機能を持ち、現在取り組むべきタスクを常時 1 件表示する。  
タイマーの状態遷移とタスク実績は外部 CSV ファイルへ保存する。

## 2. 動作環境・配布形態

- 対象 OS は Windows。
- 任意のフォルダに配置して動作できること。
- 起動は `bat` ファイルから行うこと。
- アプリ本体のコードは `ps1` で実装すること。
- データ保存先は、`bat` / `ps1` と同階層配下のフォルダとする。

## 3. 画面構成

### 3.1 メイン表示

- 画面左下に常駐させる。
- 透明背景のフローティング表示を基本とする。
- 「通常の四角いウィンドウ」を見せるのではなく、各 UI 要素が Windows 画面上に浮いて見える構成とする。
- 円形タイマーは小さめに表示し、大きく画面を占有しない。
- タイマー近傍に、現在対象となるタスクを 1 件だけ常時表示する。
- タスク管理ウィンドウを開くための導線をメイン表示内に配置する。

### 3.2 タスク管理ウィンドウ

- 別ウィンドウとして表示する。
- タスクの追加ができる。
- タスクの並べ替えができる。
- タスク状態の変更ができる。
- ドラッグ操作でタスク順を入れ替えできる。

## 4. タイマー機能

### 4.1 タイマー状態一覧

| 状態 | 説明 |
| --- | --- |
| Initial | 初期状態。タイマー未開始。 |
| IdleRunning | 0 から 1 分までのカウントアップ中。 |
| IdleChoice | Idle が 1:00 に到達し、`focus` または `break` の選択待ち。タイマーは停止して待機する。 |
| FocusRunning | Focus 中のカウントアップ状態。 |
| BreakRunning | Break 中のカウントダウン状態。 |
| Paused | 一時停止状態。停止前の状態を保持する。 |

### 4.2 初期状態

- 円メーターは空の状態で表示する。
- 時刻表示は `00:00` とする。
- 実行可能な主操作は `再生`。
- `一時停止` と `リセット` を用意する。
- `リセット` は常に初期状態へ戻す操作とする。

### 4.3 Idle

- `再生` ボタン押下で Idle を開始する。
- Idle は 0 からのカウントアップとする。
- 1 分で円メーターが満タンになる。
- 1:00 到達時に Windows のトースト通知を表示する。
- 1:00 到達時点でタイマーは停止し、`focus` または `break` の選択待ちになる。
- `focus` / `break` の選択 UI は、トースト通知とは別にアプリ上のボタンとして表示する。

### 4.4 Focus

- Idle の選択待ち状態で `focus` を押すと Focus に遷移する。
- Focus は Idle の続きとしてカウントアップを継続する。
- Focus 開始時点の表示時刻は `01:00`。
- Focus 中は時間表示が増え続ける。
- Focus 中の円メーターは満タンのまま固定する。
- Focus 中は `break` ボタンのみ表示する。
- Focus 中に `break` を押すと 5 分休憩へ遷移する。

### 4.5 Break

- Idle の選択待ち状態で `break` を押した場合は、1 分間の Break を開始する。
- Focus 中に `break` を押した場合は、5 分間の Break を開始する。
- Break はカウントダウンで表示する。
- Break 終了後は自動で 1 分間の Idle を開始する。
- Break 終了後の Idle は 0 から再開する。

### 4.6 一時停止とリセット

- `一時停止` は、Idle / Focus / Break の進行中に現在の状態と残時間または経過時間を保持して停止する。
- 一時停止後に `再生` を押した場合は、一時停止前の状態から再開する。
- `リセット` は、どの状態からでも初期状態へ戻す。
- `リセット` 実行時は、選択待ち状態や一時停止状態も解除する。

### 4.7 通知

- Idle が 1:00 に到達した時点で Windows トースト通知を 1 回表示する。
- 通知内容は「1 分経過したこと」がわかる内容とする。
- `focus` / `break` の選択はアプリ側 UI で行う。

## 5. タスク機能

### 5.1 タスク状態

タスクは以下の 3 状態を持つ。

| 状態 | 意味 |
| --- | --- |
| Normal | 通常状態。表示対象になりうる。 |
| Stopped | 停止状態。表示対象外。 |
| Done | 完了状態。表示対象外。 |

### 5.2 表示ルール

- メイン表示では、タスク一覧のうち最上位にある `Normal` 状態のタスクを 1 件だけ表示する。
- `Stopped` と `Done` のタスクはメイン表示対象にしない。
- ユーザーが明示的に選択しなくても、「最上位の `Normal` タスク」を自動的に選択状態とみなす。
- タスク順の優先度は手動並べ替え順をそのまま使う。

### 5.3 タスク管理ウィンドウでできること

- 任意の文字列でタスクを追加する。
- タスクの順序をドラッグで入れ替える。
- タスク状態を `Normal` / `Stopped` / `Done` に変更する。
- メイン表示の対象タスクは、変更後の一覧に応じて即時更新する。

## 6. 実績記録ルール

- 実績は別 CSV ファイルへ追記保存する。
- 実績として記録する対象は、「選択状態かつ Focus 状態にあるタスク」とする。
- 選択状態とは、「タスク一覧のうち最上位の `Normal` タスク」であることを指す。

### 6.1 実績を追加するタイミング

- Idle の選択待ちから `focus` に入った時点で、選択状態のタスクがあれば実績を 1 件追加する。
- Focus 中にタスク一覧の並びや状態変更が起こり、選択状態のタスクが別のタスクへ切り替わった場合は、その切替時点で新しい実績を 1 件追加する。
- Focus を一時停止後、`再生` によって Focus を再開した場合は、その再開時点を新しい開始として実績を 1 件追加する。
- Focus 中に選択対象となる `Normal` タスクが存在しない場合は、実績を追加しない。

### 6.2 実績データの意味

- 実績は「その時刻から、そのタスクに Focus を向け始めた」ことを表す開始ログとして扱う。
- 例: `10:00 タスクA`

## 7. データ保存仕様

### 7.1 保存先

- アプリと同階層配下にデータ保存用フォルダを置く。
- 例: `data\`

### 7.2 タスク一覧 CSV

ファイル名の想定: `data\tasks.csv`

必須列:

| 列名 | 内容 |
| --- | --- |
| `id` | タスクの一意識別子 |
| `order` | 並び順。小さいほど上位 |
| `title` | タスク本文 |
| `status` | `Normal` / `Stopped` / `Done` |
| `created_at` | 作成日時 |
| `updated_at` | 更新日時 |

### 7.3 実績 CSV

ファイル名の想定: `data\task_log.csv`

必須列:

| 列名 | 内容 |
| --- | --- |
| `started_at` | 実績の開始日時 |
| `task_id` | 対象タスク ID |
| `task_title` | 記録時点のタスク名 |
| `trigger` | 記録契機。例: `focus_start`, `task_switched` |

## 8. 操作フロー

### 8.1 基本フロー

1. ユーザーが `再生` を押す。
2. Idle が 0 から始まり、1 分で満タンになる。
3. 1:00 到達でトースト通知を出し、タイマーは停止して `focus` / `break` 選択待ちになる。
4. `focus` を選ぶと Focus カウントアップを開始する。
5. Focus 中は最上位 `Normal` タスクを表示し、そのタスクを選択状態として扱う。
6. Focus 開始時、選択状態のタスクがあれば実績を記録する。
7. Focus 中に `break` を押すと 5 分 Break に入る。
8. Break 終了後は自動で Idle へ戻る。

### 8.2 Idle から短い休憩を選ぶフロー

1. Idle が 1:00 に達する。
2. `break` を選ぶ。
3. 1 分 Break が始まる。
4. Break 終了後、自動で次の Idle を開始する。

## 9. 非機能要件

- 常駐利用しやすいよう、メイン表示は小さく邪魔にならないこと。
- 透明背景を利用し、視覚的には各 UI 要素のみが浮いて見えること。
- 起動場所に依存せず、相対パスでデータフォルダを参照できること。
- アプリ終了後に再起動しても、タスク一覧と過去実績が保持されること。

## 10. 補足前提

- 本仕様では、メイン表示を常に前面寄りで扱うことを前提にする。
- タスクの編集や削除は必須要件には含めない。まずは「追加・並べ替え・状態変更」を優先実装対象とする。
- Break 中の円メーター表示方法は、残り時間が把握できるカウントダウン表現を前提とする。
- 実装技術の詳細選定は未固定だが、`ps1` から扱える GUI 方式を採用する。

## 11. 受け入れ確認観点

- `bat` から起動できる。
- メイン表示が画面左下に小さく表示される。
- メイン表示が透明背景のフローティング見た目になっている。
- `再生` で Idle が始まり、1 分で満タンになる。
- 1:00 到達時にトースト通知が出る。
- 1:00 到達時に `focus` / `break` ボタンがアプリ上に出る。
- `focus` を選ぶと 1:00 から継続カウントアップし、円メーターは満タン固定になる。
- Focus 中は `break` ボタンのみ表示される。
- Idle 選択待ちからの `break` は 1 分、Focus 中の `break` は 5 分である。
- Break 終了後に自動で Idle へ戻る。
- 一時停止後に再生すると、停止前の状態から再開する。
- リセットで初期状態へ戻る。
- タスクは別ウィンドウで追加・並べ替え・状態変更ができる。
- メイン表示には最上位の `Normal` タスクだけが出る。
- `Stopped` と `Done` はメイン表示に出ない。
- タスク一覧が CSV に保存される。
- Focus 開始時と Focus 中の選択タスク切替時に、実績が CSV に追記される。
