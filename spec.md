# focus-timer 仕様書

## 1. 概要

`focus-timer` は Windows 上で動作する常駐型の軽量タイマー兼タスク管理ツールである。  
起動は `bat` ファイルから行い、アプリ本体は `ps1` で実装する。

画面左下付近に置いて使うことを前提とし、透明背景のフローティング UI とする。  
タイマーは小さな円形メーターで表示し、常時 1 件のタスクを併設表示する。

## 2. 動作環境

- 対象 OS は Windows
- 起動ファイルは `focus-timer.bat`
- アプリ本体は `focus-timer.ps1`
- データ保存先はアプリと同階層の `data\` フォルダ

## 3. メイン UI

### 3.1 基本構成

- メイン UI は透明背景のフローティング表示とする
- メイン画面は任意の場所をドラッグして移動できる
- 1 段目の左にタイマー、右に現在表示中タスクのカードを配置する
- 2 段目に操作ボタン群を横並びで配置する
- ボタンはタスクカードの直下ではなく、1 段目全体の下に配置する
- タイマーは特に小さめに表示し、全体を詰めた表示にする
- クリック要素は文字ラベルではなく、絵文字または記号を使う

### 3.2 メイン UI の表示要素

- 円形メーター
- 経過時間または残り時間のテキスト
- 再生ボタン
- 一時停止ボタン
- リセットボタン
- `focus` 選択ボタン
- `break` 選択ボタン
- `focus` 中の `break` ボタン
- 現在表示対象のタスク名
- タスク管理ボタン

### 3.3 状態名の表示

- `idle` などの内部状態名はメイン UI に表示しない
- 状態はボタン表示、時刻表示、メーター表示だけで把握できるようにする

## 4. タイマー仕様

### 4.1 タイマー状態

内部状態として以下を持つ。

| 状態 | 内容 |
| --- | --- |
| Initial | 初期状態。タイマー未開始 |
| IdleRunning | 0 から 1 分までのカウントアップ中 |
| IdleChoice | Idle が 1:00 に達し、`focus` / `break` の選択待ち |
| FocusRunning | Focus 中のカウントアップ |
| BreakRunning | Break 中のカウントダウン |
| Paused | 一時停止中 |

### 4.2 初期状態

- 時刻表示は `00:00`
- メーターは空
- 表示ボタンは `再生` `一時停止` `リセット`
- `focus` / `break` 選択ボタンは非表示

### 4.3 Idle

- 再生ボタン押下で Idle を開始する
- Idle は 0 からカウントアップする
- 1 分で円メーターが満タンになる
- 1:00 到達時に Windows トースト通知を 1 回表示する
- 1:00 到達時点でタイマーは停止し、`focus` または `break` の選択待ちになる
- この選択 UI はトースト通知ではなくアプリ上のボタンで表示する

### 4.4 Focus

- Idle の選択待ち状態で `focus` を押すと Focus を開始する
- Focus はカウントアップを継続する
- Focus 中は円メーターを満タンのまま表示する
- Focus 中は `break` ボタンのみ表示する
- Focus 中に `break` を押すと 5 分休憩へ移行する

### 4.5 Break

- Idle の選択待ち状態で `break` を押した場合は 1 分休憩を開始する
- Focus 中に `break` を押した場合は 5 分休憩を開始する
- Break はカウントダウンで表示する
- Break 終了後は自動で 1 分の Idle に戻る
- Break 終了後の Idle は 0 から開始する

### 4.6 一時停止とリセット

- 一時停止ボタンは Idle / Focus / Break の進行を止める
- 一時停止後に再生すると、停止時点から再開する
- リセットはどの状態からでも初期状態へ戻す
- リセットした場合のみ経過は破棄される

### 4.7 通知

- Idle が 1:00 に達した時点で Windows トースト通知を出す
- 通知内容は「1 分経過したこと」が分かる文面とする
- `focus` / `break` の選択はアプリ上のボタンで行う

## 5. タスク仕様

### 5.1 タスク状態

各タスクは以下の 3 状態を持つ。

| 状態 | 内容 |
| --- | --- |
| Normal | 通常状態。表示対象候補になる |
| Stopped | 停止状態。表示対象外 |
| Done | 完了状態。表示対象外、完了リストへ移る |

### 5.2 表示対象タスク

- メイン UI には常に 1 件だけタスクを表示する
- 表示対象は、並び順が最上位の `Normal` タスクとする
- `Stopped` と `Done` のタスクはメイン UI に表示しない
- `Normal` タスクが 1 件もない場合は、タスクなしのメッセージを表示する
- タスクの選択という概念は設けない
- Task Manager の一番上に来るタスクは、メイン UI の表示対象タスクと一致させる
- `Stopped` タスクは `Normal` タスクより上には表示しない
- 一番上の表示対象タスクは、他のタスクより少し強調したデザインにする

### 5.3 並び順

- 並び順は手動で管理する
- ドラッグでの並び替えは行わない
- 優先順位移動ボタンは各タスク枠の外の右側に配置する
- 利用するボタンは `一番上へ` `上へ` `下へ` `一番下へ`
- 並び替え対象は `Done` 以外のタスクのみとする
- `Done` タスクは完了リスト側へ移動し、アクティブタスクの並び順から外す

## 6. Task Manager 仕様

### 6.1 基本構成

- Task Manager は別ウィンドウで開く
- タスク追加欄を上部に配置する
- その下にアクティブタスク一覧を表示する
- さらにその下に完了タスク一覧を表示する
- 各タスク行は縦幅を最小限に抑えたコンパクトな表示にする
- 各タスクの状態文字は表示しない
- クリック要素は絵文字または記号で表現する

### 6.2 タスク追加

- 入力欄に任意の文字列を入力してタスクを追加できる
- `+` ボタン押下で追加できる
- Enter キー押下でも追加できる
- 空文字または空白のみの入力は追加しない
- 新規タスクは現在一番上にある表示対象タスクの直下へ追加する
- `Normal` タスクが 1 件もない場合は、新規タスクをアクティブ一覧の先頭へ追加する
- 完了タスクがある場合でも、新規タスクは完了一覧より前のアクティブ領域へ追加する

### 6.3 アクティブタスク一覧

- `Normal` と `Stopped` のタスクを表示する
- 各タスク行の中には停止トグルボタンと完了ボタンを置く
- 優先順位移動ボタン 4 種類は各タスク枠の外の右側に置く
- 停止トグルは `Normal` と `Stopped` を切り替える
- 完了ボタンを押すとタスクはアクティブ一覧から消え、完了一覧へ移動する
- 完了後はメイン UI の表示対象から外れる

### 6.4 完了タスク一覧

- `Done` タスクを表示する
- 完了タスクはアクティブタスク一覧とは別領域で管理する
- 完了タスクには状態文字を表示しない
- 完了タスクはメイン UI の表示対象にならない

## 7. タスク実績ログ

### 7.1 記録条件

実績は「タスクが表示対象であり、かつ Focus 状態であるとき」に記録する。

- Idle の選択待ちから `focus` に入った瞬間に、表示対象タスクがあれば 1 件記録する
- Focus 中に表示対象タスクが別タスクへ切り替わった場合は、その時点でも新しい実績を 1 件記録する
- 一時停止後に再生して Focus に戻った場合も、表示対象タスクがあれば再度記録する
- 表示対象タスクが存在しない場合は記録しない

### 7.2 記録内容

- 開始時刻
- タスク ID
- タスク名
- 記録理由

## 8. データ保存

### 8.1 保存先

- 保存先はアプリと同階層の `data\` フォルダ

### 8.2 タスク一覧 CSV

ファイル名: `data\tasks.csv`

| 列名 | 内容 |
| --- | --- |
| `id` | タスク ID |
| `order` | 並び順 |
| `title` | タスク名 |
| `status` | `Normal` / `Stopped` / `Done` |
| `created_at` | 作成日時 |
| `updated_at` | 更新日時 |

### 8.3 実績ログ CSV

ファイル名: `data\task_log.csv`

| 列名 | 内容 |
| --- | --- |
| `started_at` | 実績記録時刻 |
| `task_id` | タスク ID |
| `task_title` | タスク名 |
| `trigger` | 記録契機。例: `focus_start`, `task_switched`, `focus_resumed` |

## 9. ボタン表現

- クリック要素は絵文字または記号で表す
- テキストラベルを前面に出したボタンは使わない
- ツールチップで意味を補足する
- 見た目はフラットでモダンな方向性にする
- 強い影、光沢、過剰な枠線は避ける

## 10. テスト方針

- タイマー状態遷移は単体テスト対象とする
- タスク追加、並び替え、停止トグル、完了遷移は単体テスト対象とする
- CSV の保存と読込は単体テスト対象とする
- Enter によるタスク追加はスモークテスト対象とする
- UI 配置の主要ルールは XAML スモークテスト対象とする
- GUI の実クリックや Windows トーストの見た目そのものは手動確認対象とする
